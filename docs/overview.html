Connected components

  • MCP3008 ADC on SPI0. CH0 = Sensor 1, CH1 = Sensor 2. Both sensors are placed in the same plant pot.
  • 4-channel relay module JQC-3FF-S-Z 5V, active-LOW. IN1↔K1↔Pump 1, IN2↔K2↔Pump 2. IN3 and IN4 are unused (IN4 floats LOW so its LED stays lit — harmless).
  • Pumps: 9V battery → through relay NO/COM contacts → pump motor. Pump 1 is dead downstream of the relay; only Pump 2 actually moves water.
  • SSD1306 OLED at I²C 0x3C. Severe burn-in from the previous moisture-graph display. Will not be ported.
  • Capacitive moisture sensors powered through a transistor gate so they spend most of their life unpowered (corrosion mitigation + lower idle current).

Gotchas

active-low
All control outputs (sensor gate, both relay INs) are active-LOW. Drive HIGH at boot to keep pumps off and sensors unpowered.
psu
Under-voltage and throttling flags are set (get_throttled = 0x50000). A better PSU or a separate supply for the relay/pumps would be wise before rewiring more current onto this rail.
spi
CS is software-driven on GPIO 5, not the hardware CE0/CE1. Keep that in mind when porting code.

Free pins (for later)

  • D3 — GPIO 0, strapping pin; tied to FLASH button. Leave alone unless you have a specific input use and an external pull-up.
  • D4 — GPIO 2, strapping pin; shares the onboard blue LED. Usable as an output if you accept a brief blink at boot.
  • D8 — GPIO 15, also a strapping pin (must idle LOW); usable as plain output if you mind the boot behaviour.
  • A0 — the only analog input, 0–3.3 V via the onboard voltage divider. Free in this build (MCP3008 handles the actual sensor reads).
  • RX / TX — used for serial debug.

New firmware (what the ESP will do)

  • Read the MCP3008 with pulsed sampling (median-of-medians) — same robustness pattern as the Pi’s PotPi v2.
  • Persist readings to LittleFS on the ESP’s 4 MB flash.
  • Connect to a fixed WiFi network.
  • Push daily CSV files to a GitHub repo via the Contents API (HTTPS PUT).
  • A GitHub Pages site in the same repo renders the data using Chart.js.

Gotchas

strapping pins
GPIO 0 (D3), GPIO 2 (D4) and GPIO 15 (D8) are strapping pins — their levels are sampled during the ~150 ms boot window to choose boot mode. If a load (the relay’s opto-input or the sensor-power transistor) holds them at the wrong level at reset, the chip can boot into UART download mode instead of running our firmware, or the relay can twitch on every power-up. Our outputs have been moved off these pins (sensor gate → D2 / GPIO 4, pump relay → D1 / GPIO 5).
pull-up
Add an external 10 kΩ pull-up from the relay IN line to 3V3 so the line is guaranteed HIGH (relay OFF) before our firmware sets the pin as output. A second 10 kΩ pull-up on the sensor-gate line is recommended for the same reason.
active-low
Sensor gate and relay IN2 are both active-LOW. Initialise GPIOs HIGH before setting them as outputs to avoid a momentary pulse.
d0 / gpio 16
GPIO 16 has no interrupt and no PWM, and it’s on a different silicon block. Plain digital output (CS) is fine; don’t pick it for anything time-sensitive.
power
Pump still runs from the 9V battery via the relay contacts; the relay coil takes 5V from Vin (USB). Do not feed the pump rail into the MCU’s 3V3.

What changes, what stays

Stays the same

  • MCP3008 ADC — same chip, same SPI mode, same channel assignment (CH0 = Sensor 1, CH1 = Sensor 2).
  • Capacitive moisture sensors — same pair, same pot, same calibration constants carry over (re-verify after rewiring).
  • Pump 2 + mini-pump motor — unchanged.
  • 9V battery as the pump supply, switched through relay K2’s NO/COM.
  • JQC-3FF-S-Z 4-channel relay module, still active-LOW.
  • Pulsed-sampling pattern from PotPi v2 (median-of-medians).
  • Sensor power gating via a transistor — same idea, different MCU pin.

Changes

  • Host board: Raspberry Pi 3B → NodeMCU ESP8266 (Amica v1.0).
  • Pin numbers: BCM GPIOs → NodeMCU D-labels / ESP GPIOs (see tables above).
  • CS pin: GPIO 5 (Pi) → D0 / GPIO 16 (ESP) — still software-controlled.
  • Sensor power gate: GPIO 18 → D2 / GPIO 4 (moved off strapping pins; add a 10 kΩ pull-up).
  • Pump 2 control: GPIO 17 → D1 / GPIO 5 (moved off strapping pins; add a 10 kΩ pull-up on the relay IN line).
  • OLED display removed — SSD1306 stays with the Pi.
  • Pump 1 stays unwired in the new build (motor still dead).
  • Storage: SD card filesystem → LittleFS on ESP flash.
  • Connectivity: local-only on Pi → WiFi + GitHub Contents API, rendered by Chart.js on GitHub Pages.
  • Relay coil supply: from Pi 5V rail → from NodeMCU Vin (USB pass-through).
